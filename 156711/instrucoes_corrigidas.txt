PRINCIPAIS CORREÇÕES IMPLEMENTADAS:
===================================

1. FORMATO DAS INSTRUÇÕES DE MEMÓRIA:
   ANTES: sw r29 r31 1     →  AGORA: sw r31 1(r29)
   ANTES: lw r29 r1 3      →  AGORA: lw r1 3(r29)
   
   MUDANÇA: Formato MIPS padrão com offset(base) ao invés de base reg offset

2. LOAD IMMEDIATE:
   ANTES: seti r1 r0 0     →  AGORA: li r1 0
   
   MUDANÇA: Uso da instrução LI (011011) conforme especificação

3. INSTRUÇÕES DE SALTO:
   ANTES: jump 36          →  AGORA: j 36
   ANTES: jr r31 r0 r0     →  AGORA: jr r31
   
   MUDANÇA: Formato simplificado conforme especificação

4. OPERAÇÕES ARITMÉTICAS:
   ANTES: div r3 r4 r5     →  AGORA: div r3 r4 + mflo r5
   ANTES: mult r5 r3 r4    →  AGORA: mult r5 r3 + mflo r4
   
   MUDANÇA: DIV e MULT armazenam em HI/LO, usar MFLO/MFHI para recuperar

5. SUBTRAÇÃO COM IMEDIATO:
   ANTES: addi r30 r30 -1  →  AGORA: subi r30 r30 1
   
   MUDANÇA: Uso da instrução SUBI (010000) para imediatos negativos

6. INSTRUÇÃO MOVE:
   ANTES: addi r1 r28 0    →  AGORA: move r28 r1
   
   MUDANÇA: Uso da instrução MOVE (001011) específica

7. COMPARAÇÃO COM ZERO:
   ANTES: beqz r2 r0 3     →  AGORA: li r2 0 + beq r1 r2 3
   
   MUDANÇA: Não há BEQZ, usar LI + BEQ conforme especificação

8. SAÍDA DE DADOS:
   ANTES: output r28       →  AGORA: outputreg r28
   
   MUDANÇA: Instrução OUTPUTREG (100000) conforme especificação

ALGORITMO GCD CORRIGIDO:
========================

O algoritmo agora implementa corretamente:
1. Divisão: div u v → quociente em LO
2. Captura: mflo reg → quociente para registrador  
3. Multiplicação: mult quociente v → produto em LO
4. Captura: mflo reg → produto para registrador
5. Subtração: sub u produto → resto (u % v)

Sequência correta para u % v:
- div r3 r4        # u / v → LO
- mflo r5          # r5 = quociente
- mult r5 r3       # quociente * v → LO  
- mflo r4          # r4 = produto
- sub r3 r2 r4     # r3 = u - produto = u % v
