PROBLEMA CRÍTICO DESCOBERTO E RESOLVIDO
===============================================================================

🚨 REGRESSÃO SÉRIA: Memory corruption em test_array_access.c
SINTOMA: 
- Resultado incorreto: [200, 144, 44, 144] 
- Resultado esperado: [200, 400, 300, 400]
- Corrupção específica: MEM[156] alterado de 300 para 44

ANÁLISE DA CAUSA RAIZ:
- Versão Testing tinha lógica adicional para arrays de parâmetros
- Condição problemática: `else if (array_sym->is_arg && array_sym->is_array)`
- Arrays locais sendo processados incorretamente como arrays de parâmetros
- Endereço base incorreto gerado para arrays locais

DEBUGGING DETALHADO:
- Ciclo 50: MEM[156] = 300 (correto)
- Ciclo 96: MEM[156] = 44 (corrompido)
- Registrador R4 com cálculo de endereço incorreto
- Problema na reutilização de registradores temporários

===============================================================================
SOLUÇÃO IMPLEMENTADA
===============================================================================

CORREÇÃO CIRÚRGICA em process_array_access (assembler.c):

ANTES (problemático):
```c
if (array_sym->is_global) {
    // Array global: GP + offset + (index * 4)
} else if (array_sym->is_arg && array_sym->is_array) {
    // Array parâmetro: carregar endereço base do parâmetro
} else {
    // Array local: FP + offset + (index * 4)
}
```

DEPOIS (corrigido):
```c
if (array_sym->is_global) {
    // Array global: GP + offset + (index * 4)
} else if (array_sym->is_arg && array_sym->is_array && !array_sym->is_global) {
    // Array parâmetro: carregar endereço base do parâmetro
} else {
    // Array local: FP + offset + (index * 4)
}
```

DIFERENCIAL CRÍTICO: Adição da condição `&& !array_sym->is_global` para 
distinguir corretamente arrays de parâmetros de arrays locais.
