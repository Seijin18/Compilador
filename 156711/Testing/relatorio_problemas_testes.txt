```c
int numbers[5];
void main(void) {
    int i; int sum;
    i = 0;
    while (i < 5) {
        numbers[i] = (i + 1) * 10;  /* 10, 20, 30, 40, 50 */
        i = i + 1;
    }
    i = 0; sum = 0;
    while (i < 5) {
        sum = sum + numbers[i];
        i = i + 1;
    }
    output(sum);  /* Deve imprimir 150 (10+20+30+40+50) */
    i = 0;
    while (i < 3) {
        output(numbers[i]);
        i = i + 1;
    }
}
```

1. ASSEMBLER - Bug crítico na condição de loop while:
   - Condição while (i < 5) não está sendo avaliada corretamente
   - Variável i começou em 0, mas chegou ao valor 49 no final da execução
   - Controle de fluxo completamente quebrado

2. ASSEMBLER - Mesmo problema ADDI R0 persistente:
   - Bug sistemático "ADDI R0, GP, 0" presente em todas as operações de array
   - Confirma problema fundamental não resolvido no assembler corrigido

3. GRAVE - Condições de branch incorretas:
   - BEQ R4, R0 sempre falha porque R0 = 156 (GP) e R4 = 0
   - Condição de saída do loop nunca é satisfeita
   - SLT e BEQ não estão trabalhando juntos corretamente

4. CRÍTICO - Loop infinito em programas com while:
   - Qualquer programa com loops while será executado indefinidamente
   - Sistema não consegue processar condições de parada
   - Timeout necessário para interromper execução

IMPACTO: CRÍTICO - Loops while completamente quebrados, sistema inutilizável para programas iterativos

TESTE 5: test_multiple_functions.c
===============================================================================

CÓDIGO FONTE:
```c
int add(int a, int b) { return a + b; }
int multiply(int x, int y) { return x * y; }
int calculate(int numa, int numb) {
    int sumresult; int multresult;
    sumresult = add(numa, numb);
    multresult = multiply(sumresult, 2);
    return multresult;
}
void main(void) {
    int result;
    result = calculate(3, 7);
    output(result);
}
```

RESULTADO ESPERADO: 20 (calculate(3,7) = multiply(add(3,7), 2) = multiply(10, 2) = 20)
RESULTADO OBTIDO: LOOP INFINITO (execução interrompida após detecção de loop)

PROBLEMAS IDENTIFICADOS:

1. COMPILADOR - Rejeita identificadores com números:
   - Erro léxico com 'num1', 'num2' no código original
   - Problema: analisador léxico não aceita dígitos em identificadores
   - Solução temporária: renomear para 'numa', 'numb'

2. CRÍTICO - Passagem de parâmetros entre funções falha completamente:
   - Parâmetros 3 e 7 não chegam nas funções add() e multiply()
   - Todas as operações são feitas com valores 0
   - Resultado: add(0,0)=0, multiply(0,2)=0, saída final=0

3. GRAVE - Problema na inicialização de Frame Pointer:
   - Parâmetros são carregados de endereços de memória vazios (0)
   - LW R7, 0(FP) e LW R8, 1(FP) retornam 0 sempre
   - Sistema de pilha não está passando argumentos corretamente

4. CRÍTICO - Return address corrompido causa loop infinito:
   - RA (Return Address) fica como 0 após chamadas de função
   - JR RA salta para PC=0, reiniciando o programa
   - Sistema detecta loop e interrompe execução

5. ASSEMBLER - Gerenciamento de pilha incorreto:
   - Frame Pointer não aponta para posições corretas dos parâmetros
   - Stack Pointer manipulation está incorreta
   - Contexto de função se perde entre chamadas

IMPACTO: CRÍTICO - Chamadas de função completamente quebradas, sistema inutilizável para programas modulares
